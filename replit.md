# Cloud-Surgeon

Autonomous AI DevOps agent — detects, diagnoses, and repairs cloud infrastructure incidents. Built for the CockroachDB × AWS Hackathon 2026.

## Architecture

pnpm monorepo with two runnable artifacts:

- **`artifacts/api-server`** — Node.js/Express backend (port 8080, preview path `/api`)
- **`artifacts/dashboard`** — React/Vite frontend (preview path `/dashboard/`)

Shared libraries under the workspace root:
- `packages/db` — Drizzle ORM schema + CockroachDB client
- `packages/api-zod` — Zod-validated API types (generated by Orval from `openapi.yaml`)
- `packages/api-client-react` — React Query hooks (generated by Orval)
- `packages/integrations-anthropic-ai` — Anthropic AI integration proxy

## How to run

Both workflows start automatically. To restart manually:

```bash
# API server (builds then starts)
pnpm --filter @workspace/api-server run dev

# Dashboard
pnpm --filter @workspace/dashboard run dev
```

## Key environment variables

| Variable | Purpose |
|---|---|
| `COCKROACHDB_URL` | CockroachDB connection string |
| `CLOUD_SURGEON_API_KEY` | Shared API key (dashboard ↔ API) |
| `AI_PROVIDER` | `mistral` (default) or `anthropic` |
| `BEDROCK_API_KEY` | Bedrock API key for Mistral via bedrock-mantle |
| `VOYAGE_API_KEY` | Voyage AI for semantic embeddings (falls back to hash) |
| `AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY` | Live AWS tool calls (ECS, RDS, Lambda, CloudWatch) |
| `COCKROACH_CLOUD_API_KEY` | CockroachDB Cloud API (cluster management) |
| `DASHBOARD_PASSWORD` | Password protecting the React dashboard |

All secrets are managed via Replit Secrets (never commit them).

## Dashboard login

The dashboard at `/dashboard/` is password-protected. The password hint is shown on the login screen and comes from `VITE_DASHBOARD_PASSWORD` (set at build time) or `DASHBOARD_PASSWORD`.

## Notes

- AWS actions are **simulated by default** when no AWS credentials are present — safe for demo.
- `AI_PROVIDER=mistral` routes through `bedrock-mantle.ts` (OpenAI-compat Bearer auth with `BEDROCK_API_KEY`).
- CockroachDB DDL uses hand-written SQL rather than `drizzle-kit push` — see `cockroachdb-serverless-connection.md` in `.agents/memory/`.
- The `.tools/ccloud` binary is used for CockroachDB Cloud CLI calls; REST API is the fallback.
