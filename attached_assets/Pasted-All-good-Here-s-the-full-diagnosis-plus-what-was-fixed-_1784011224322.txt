The artifact's preview path is /api (from artifact.toml), not /api-server. So if the changefeed were ever dropped and recreated it would point at a 404. Fixed — URL is now /api/internal/cdc. The reused changefeed from the previous session was created with the correct URL, which is why CDC has been working; the bug was latent.
3 — anomaly.test.ts failures (fixed, 67/67 pass)

Root cause: IncidentContext in openapi.yaml already has additionalProperties: true, but orval's Zod generator silently ignores it — it emits zod.object({...}) without .passthrough(). Predictive incidents write extra fields into contextJson (source, predictiveMetric, detectionMethod, similarityScore) that Zod was stripping on parse.

Added .passthrough() to the contextJson object in the three affected generated schemas (TriggerIncidentResponse, ListIncidentsResponseItem, GetIncidentResponse), with a comment marking the manual override so the next orval regeneration doesn't silently drop it without notice.
